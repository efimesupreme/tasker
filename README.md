# Проектный трекер

Проектный трекер — статическое веб-приложение для ведения личных и рабочих проектов. Внутри можно создавать проекты, разбивать их на группы задач и подзадачи, отмечать выполненные шаги, видеть ближайшие доступные действия и находить элементы без движения 14+ дней.

Приложение подготовлено для публикации через GitHub Pages: сборка, серверная часть и база данных не нужны.

## Что внутри

- `index.html` — HTML-разметка интерфейса и подключение статических ресурсов.
- `styles.css` — адаптивная тёмная тема без внешних CSS-библиотек.
- `app.js` — демо-данные, рендеринг интерфейса, базовые CRUD-формы, расчёт прогресса, работа с зависимостями и сохранение в браузере.

## Как открыть локально

Самый простой способ — открыть `index.html` двойным кликом в браузере. Приложение не обращается к серверу и работает как обычный статический сайт.

Если нужно проверить поведение ближе к GitHub Pages, запустите статический сервер из корня проекта:

```bash
python3 -m http.server 8000
```

После запуска откройте <http://localhost:8000>.

## Как опубликовать через GitHub Pages

1. Загрузите файлы `index.html`, `styles.css`, `app.js` и `README.md` в GitHub-репозиторий.
2. Откройте настройки репозитория: **Settings → Pages**.
3. В блоке **Build and deployment** выберите источник **Deploy from a branch**.
4. Выберите ветку, например `main`, и папку `/root`.
5. Нажмите **Save** и дождитесь публикации.
6. Откройте ссылку вида `https://<user>.github.io/<repository>/`.

Проект использует относительные пути `./styles.css` и `./app.js`, поэтому его можно публиковать как в корне домена, так и в подпапке репозитория GitHub Pages.

## Где хранятся данные

Данные хранятся локально в браузере в `localStorage` под ключом:

```text
tasker.projects.v1
```

Это значит, что данные:

- остаются на устройстве пользователя;
- не отправляются на сервер;
- не синхронизируются между браузерами и устройствами;
- могут быть очищены вместе с данными сайта в настройках браузера.

При первом запуске или при отсутствии сохранённых данных приложение показывает демо-проекты из `app.js`.

## Как сделать резервную копию

1. Откройте опубликованное или локальное приложение в браузере.
2. Откройте инструменты разработчика: `F12` или `Ctrl+Shift+I` / `Cmd+Option+I`.
3. Перейдите во вкладку **Console**.
4. Выполните команду:

```js
copy(localStorage.getItem('tasker.projects.v1'))
```

5. Вставьте скопированный JSON в файл, например `tasker-backup.json`, и сохраните его в надёжном месте.

Если команда `copy` недоступна в вашем браузере, выполните:

```js
localStorage.getItem('tasker.projects.v1')
```

Затем вручную скопируйте выведенную строку.

## Как восстановить данные из JSON

1. Откройте приложение в браузере.
2. Откройте инструменты разработчика и вкладку **Console**.
3. Подготовьте JSON из резервной копии. Он должен быть строкой с массивом проектов.
4. Выполните команду, заменив `PASTE_BACKUP_JSON_HERE` содержимым резервной копии:

```js
localStorage.setItem('tasker.projects.v1', JSON.stringify(JSON.parse(`PASTE_BACKUP_JSON_HERE`)))
location.reload()
```

Если резервная копия уже находится в буфере обмена, удобнее временно вставить её в переменную:

```js
const backup = `PASTE_BACKUP_JSON_HERE`
localStorage.setItem('tasker.projects.v1', JSON.stringify(JSON.parse(backup)))
location.reload()
```

После перезагрузки приложение прочитает данные из `localStorage`.

## Ограничения первой версии

- Нет серверной базы данных, авторизации и синхронизации между устройствами.
- Данные зависят от конкретного браузера и профиля пользователя.
- Импорт и экспорт JSON пока выполняются вручную через инструменты разработчика.
- Нет автоматических напоминаний, календаря и уведомлений.
- Разделы «Архив» и «Настройки» пока являются заглушками.
- Поиск и кнопка «Фильтры» в интерфейсе подготовлены визуально, но расширенная фильтрация ещё не реализована.
- История изменений и отмена действий не сохраняются.
